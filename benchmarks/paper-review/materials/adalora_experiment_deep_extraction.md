## 1. 实验目标与作者想验证的核心结论

### 核心结论 1：AdaLoRA 自适应预算分配优于均匀预算分配方法

- **对应实验**：Table 1（GLUE）、Table 2（SQuAD）、Table 3（XSum / CNN/DailyMail）
- **证据**：在三个 benchmark 族（NLU、QA、NLG）上 AdaLoRA 在所有预算水平下均优于或持平 LoRA、Adapter、BitFit 等均匀分配方法；在低预算设置下增益尤为显著（Section 1, Page 1）

### 核心结论 2：SVD 参数化可有效实现秩剪枝，无需昂贵的精确 SVD 计算

- **对应实验**：Table 5（SVD-LoRA vs LoRA ablation）、Table 4（AdaLoRA vs Prune LoRA）、Table 15（训练代价对比）
- **证据**：SVD-LoRA（仅 SVD 参数化，无自适应预算分配）在多数设置下优于原始 LoRA；AdaLoRA 仅增加 11-16% 训练时间（Table 15），远低于精确 SVD 的 O(min(d1,d2)d1d2) 复杂度（Section 3.1, Page 4）

### 核心结论 3：基于敏感性和不确定性的重要性指标优于朴素奇异值幅度指标

- **对应实验**：Table 4（不同重要性指标变体对比）
- **证据**：论文提出的重要性指标（公式 7 + 公式 11）在 SST-2、RTE、CoLA 三个数据集上普遍优于仅使用奇异值幅度 Si = |λi| 或仅使用敏感性 s(·) = I(·)；性能退化最高达 0.9%（Section 4.4, Page 9）

### 核心结论 4：AdaLoRA 在极低预算下表现尤为突出

- **对应实验**：Table 1（GLUE 0.3M）、Table 2（SQuAD 0.08%）、Table 3（NLG 0.13%）
- **证据**：AdaLoRA 以 0.3M 参数在 CoLA 上达到 70.04 Mcc，超越所有基线方法在更高预算（0.6M、1.2M）下的结果（Table 1, Page 7）；在 SQuADv2.0 上以 0.08% 参数达到 88.7 F1，接近其高预算性能且比最佳基线高 1.2%（Table 2, Page 8）

### 核心结论 5：SVD 参数化与自适应预算分配两个组件对性能均有贡献

- **对应实验**：Table 5（组件消融）
- **证据**：单独使用 SVD-LoRA 优于 LoRA 但不及完整 AdaLoRA；去除正交正则化后（AdaLoRAγ=0）性能退化，验证了两个组件的必要性（Section 4.4, Page 9-10）

## 4. 消融实验提取

### 消融 1：AdaLoRA vs LoRA 结构化剪枝（Table 4 上半部分）

- **消融内容**：对比 AdaLoRA 的 SVD 参数化剪枝 vs 对原始 LoRA 双元组（doublet）进行结构化剪枝，两者使用相同的重要性指标、预算调度器和训练配置
- **测试变体**：
  - Prune LoRA：对 LoRA 的 BA 双元组进行结构化剪枝
  - AdaLoRA：对 SVD 参数化三联组进行剪枝（保留奇异向量，仅置零奇异值）
- **测试数据集**：SST-2、RTE、CoLA
- **预算水平**：0.08%、0.16%、0.65%

| 方法 | SST-2 0.08/0.16/0.65% | RTE 0.08/0.16/0.65% | CoLA 0.08/0.16/0.65% |
|------|----------------------|--------------------|--------------------|
| Prune LoRA | 94.84 / 94.50 / 94.95 | 86.28 / 86.15 / 87.00 | 66.71 / 69.29 / 69.57 |
| **AdaLoRA** | **95.52 / 95.80 / 96.10** | **87.36 / 87.73 / 88.09** | **70.21 / 70.04 / 71.45** |

来源：Table 4, Page 9

**关键发现**：
- AdaLoRA 在所有数据集和所有预算水平下均优于 Prune LoRA
- 差距在低预算下尤为显著，如 CoLA 0.08%: 70.21 vs 66.71（+3.50）
- 论文归因于 AdaLoRA 的 SVD 参数化保留奇异向量、仅置零奇异值的设计保留了未来恢复的可能性，而 LoRA 双元组一旦剪枝则完全归零难以恢复（Section 3.1, Page 4；Section 4.4, Page 9）

### 消融 2：不同重要性指标变体对比（Table 4 下半部分）

- **消融内容**：对比三种重要性指标方案
- **测试变体**：
  - s(·) = I(·)：仅使用敏感性（公式 8），不使用不确定性量化
  - Si = |λi|：仅使用奇异值幅度
  - AdaLoRA（完整）：敏感性 + 不确定性量化（公式 7 + 公式 11）
- **测试数据集**：SST-2、RTE、CoLA
- **预算水平**：0.08%、0.16%、0.65%

| 指标变体 | SST-2 0.08/0.16/0.65% | RTE 0.08/0.16/0.65% | CoLA 0.08/0.16/0.65% |
|---------|----------------------|--------------------|--------------------|
| s(·) = I(·) | 94.61 / 95.30 / 95.64 | 87.36 / 87.71 / 88.10 | 66.71 / 68.83 / 70.19 |
| Si = |λi| | 95.41 / 95.41 / 95.87 | 87.00 / 86.28 / 88.00 | 67.67 / 68.44 / 70.38 |
| **AdaLoRA（完整）** | **95.52 / 95.80 / 96.10** | **87.36 / 87.73 / 88.09** | **70.21 / 70.04 / 71.45** |

来源：Table 4, Page 9

**关键发现**：
- 完整指标在绝大多数设置下表现最佳
- 退化最高达 0.9%（如 CoLA 0.16%: 68.44 vs 70.04，退化 1.60 绝对值，但论文称"up to 0.9%"可能指其他设置）（Section 4.4, Page 9）
- Si = |λi| 在某些设置下表现尚可，但整体不如完整指标

## 8. 值得关注的实验现象

### 现象 1：低预算下 AdaLoRA 出现超越高预算基线的"降维打击"

AdaLoRA 在 0.3M 参数下在 CoLA 上达到 70.04 Mcc，不仅超越所有基线在同等预算下的结果，还超越它们在 0.6M 和 1.2M 预算下的结果。类似现象在 SQuADv2.0 上也存在——0.08% 预算的 88.7 F1 接近 0.65% 预算的 88.9 F1。这表明自适应预算分配在极端参数受限场景下尤其有效。

**来源**：Table 1, Page 7；Table 2, Page 8

### 现象 2：FFN 和顶层比 Self-Attention 和底层更重要

Figure 3 显示 FFN 的权重矩阵（Wf1, Wf2）在所有层中分配了更高的秩，顶层 Transformer 层的秩普遍高于底层。这与 Figure 1 的初步实验结果一致——"fine-tuning FFN achieves better performance than self-attention modules"和"top layers are more important than bottom layers"。

**来源**：Figure 1, Page 2；Figure 3, Page 10

### 现象 3：SVD 参数化本身的改进就已超越 LoRA

仅引入 SVD 参数化而不进行自适应预算分配（SVD-LoRA），在多数设置下已优于原始 LoRA。如在 SST-2 0.08% 上 SVD-LoRA (95.33) vs LoRA (94.38)。这可能源于正交正则化带来的训练稳定性。

**来源**：Table 5, Page 10

### 现象 4：正交正则化的去除导致 0.32% 和 0.65% 预算下性能反常下降

在 Table 5 中，AdaLoRAγ=0 在 SST-2 的 0.32% 和 0.65% 预算下分别只有 95.30 和 95.10，不仅低于 AdaLoRA 的 96.10，甚至低于 SVD-LoRA 的 95.07 和 95.53。这表明缺少正交正则化后，高预算下的 AdaLoRA 可能因非正交的奇异向量导致剪枝后的近似误差增大。

**来源**：Table 5, Page 10

### 现象 5：CNN/DailyMail 上 AdaLoRA 增益小于其他任务

在 CNN/DailyMail 上（Table 3），AdaLoRA 与 LoRA 的表现差距很小（如 2.20% 预算下 R-1: 45.00 vs 45.03，反而 LoRA 略高 0.03）。这与 XSum 上的明显差距形成对比，表明 NLG 任务对自适应预算分配的需求可能因数据集特性而异。

**来源**：Table 3, Page 8

### 现象 6：SQuADv2.0 上 LoRA 在 0.16% 预算下出现性能回退

在 SQuADv2.0 上，LoRA 的 F1 从 0.08% 预算的 87.5 下降到 0.16% 预算的 86.7，虽然参数增加但性能反而下降。而 AdaLoRA 保持了一致的性能（88.7 → 88.8）。这表明在 LoRA 中机械地增加均匀的秩可能引入噪声而非信息。

**来源**：Table 2, Page 8

## 11. 一段简短总结

AdaLoRA 在三个 benchmark 族（GLUE NLU、SQuAD QA、XSum/CNN-DailyMail NLG）、两个预训练模型（DeBERTaV3-base、BART-large）上验证了自适应参数预算分配的有效性。核心实验发现：(1) AdaLoRA 在所有 12 个数据集的绝大多数预算水平上优于 LoRA 和 Adapter 等均匀分配方法；(2) 低预算下的增益尤其显著——0.3M 参数在 CoLA 上超越 1.2M 预算的基线；(3) SVD 参数化使秩剪枝成为可能且代价可控（仅增 11-16% 训练时间）；(4) FFN 和顶层 Transformer 层获得更多预算分配。证据充分性最强的结论是 AdaLoRA > 均匀分配方法（12/12 数据集一致验证），但跨 LLM 架构泛化性（仅测试 2 个模型）、重要性指标的跨任务消融（仅 3 NLU 任务验证）、CNN/DailyMail 上的边际增益等关键问题尚未充分解答。代码和数据配置均已开源公开，为复现和下游验证提供了良好基础。
