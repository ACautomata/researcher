# Paper Demo Output Spec

The paper demo produces one final Markdown file:

```text
recommended-ideas.md
```

The file should contain:

1. Brief topic and scope
2. Processed paper list
3. Paper-by-paper summary
4. Cross-paper summary
5. Main limitations or gaps extracted from the papers
6. Recommended ideas ranked by priority
7. Open questions or assumptions
8. Human feedback prompt for the next iteration

Each recommended idea must include:

- title
- one-sentence hypothesis
- paper insight or limitation addressed
- proposed improvement
- evidence chain
- minimum validation experiment
- expected metric change
- implementation scope
- risks
- confidence
- recommendation reason

Use Markdown headings and short paragraphs. Avoid large tables.
