# extract — 论文实验深度提取

## Mission

基于 Wiki 条目和论文原文，对实验部分做结构化深化提取，产出完整 12 节 Markdown 文档（## 0–## 11），供下游 `critic`（问题分析）和 `design`（验证设计）消费。

## When to use

- 需要从论文中提炼可比较、可复用的实验信息
- 为 `critic` / `design` 准备结构化实验材料

不要用于：问题分析（`critic`）、验证设计（`design`）、论文入库（`ingest`）。

## 执行原则

- 当前阶段只做实验结果深化提取，不做问题挖掘、不提出新方法
- 不简单复述实验部分，要提炼结构化、可比较、可复用信息
- 论文未提供的信息写"论文中未明确说明"，不自行推断
- 区分"论文报告" / "间接观察" / "未提供"

## 重点关注内容

实验目标、数据集、任务/阶段划分、baseline、评价指标、主结果、消融实验、参数敏感性与稳定性、效率与复杂度、鲁棒性与泛化性、值得关注的实验现象、论文结论与实验支撑的关系。

增量学习 / 持续学习 / 动态图学习 / 节点分类场景额外关注：不同阶段性能变化、新旧知识保持与适应、任务顺序与结构变化强度下的表现、memory budget 敏感性、特定数据集退化、性能提升的代价。

## 输入

| 材料 | 必需 |
|------|------|
| Wiki 论文条目 | 是 |
| 论文原文（PDF/URL） | 推荐 |

## 输出结构

严格按以下 12 节输出为一份完整 md，先写到 `raw/sources/<slug>.md`，然后调用 `ingest`（传入该 md 文件路径）统一写入 wiki；**不直接调用 `wiki_apply` 建页**：

```markdown
# {{论文标题}}：实验结果深化提取

## 0. 文档定位
## 1. 实验目标与作者想验证的核心结论
## 2. 实验设置总览
  ### 2.1 数据集
  ### 2.2 任务/阶段划分方式
  ### 2.3 对比方法
  ### 2.4 评价指标
  ### 2.5 实现与训练相关设置
## 3. 主结果提取
  ### 3.1 主结果结论
  ### 3.2 结果差异与模式
  ### 3.3 作者结论与主结果的对应关系
## 4. 消融实验提取
## 5. 参数敏感性与稳定性分析
## 6. 效率、复杂度与资源代价
## 7. 鲁棒性、泛化性与补充实验
## 8. 值得关注的实验现象（3–6 个，只描述不批判）
## 9. 证据充分性整理
## 10. 对后续问题发现最有价值的实验信息
## 11. 一段简短总结（100–150 字）
```

## 完成门禁

- Reply 包含全部 12 节
- 未提供信息一律标注"论文中未明确说明"，不编造
- 产出 md 已经 `ingest` 写入 wiki（含 wiki 路径），且本 skill 未直接调 `wiki_apply` 建页
